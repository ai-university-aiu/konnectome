PART THREE - WHAT I DID (THE REPORT)


I built konnectome from an empty public repository to a running, integrated, glass-box Proto-Artificial-General-Intelligence (Proto-AGI) prototype that also combines thoughts, across fourteen merged pull requests.

The setup.

• I created the repository scaffold: the CONSTITUTION.md of fourteen commandments (the owner added a fifteenth mid-build), the fourteen reference documents copied into docs/, the ledger, the two change orders, the archive folder, the README in the crimson-to-gold family style with a placeholder banner, and the six-file SPARCD Fileset seeded verbatim from the appendices of Nature's Cognitive Architecture. That was pull request 1.

The ten architecture components (Section A3.2), each realized as a pack of three files (a manifest, the module, and a Prolog Unit test suite), written in English Readable Code with whole-word snake_case names, and gated.

• tick_engine - Component 2, the Scheduler and Tick Engine, with the two-pass synchronous update (the heartbeat).

• archetype - the six dynamical archetype rules (relay, integrator, oscillator, attractor, gate, comparator) and their dispatch.

• observer - Component 9, the passive Observer, which records each tick as a Causalontology token_occurrence by reusing PrologAI's causal_core (konnectome's first durable thought).

• connection_graph - Component 3, the Connection Graph: directed weighted interfaces, signal propagation one hop per tick, and a bus-modulated update whose relay gains are set by the neuromodulators.

• neuromodulator_bus - Component 4, the neuromodulatory broadcast bus.

• drive_system - Component 5, the homeostatic drives: reward as the total reduction of drive error, broadcast as dopamine, plus action proposals and the body's response to action.

• action_selector - Component 6, the referee that releases exactly one candidate action and never invents one.

• override_controller - Component 7, the safety property: a vital drive in distress seizes control, and the respiration drive can never be suppressed.

• plasticity_engine - Component 8, the three-factor learning rule (sending activity, receiving activity, dopamine) that changes connection weights.

• perturbation_interface - Component 10, the experimenter's hand: lesion, cut, and shift (for example, deplete dopamine to model Parkinson's disease).

• cognitive_cycle - the integration: it wires all ten components into one running tick in the Section A3.3 order, and completes Component 1 (the construct registry) as the world it carries. In its hardened form it is a closed sensorimotor loop: the drives propose actions, a construct's gain is set by the bus, the action is released under the safety override, the plasticity engine learns, and the body responds - so acting to reduce a need moves the body toward its set-point, and that improvement feeds back as the next tick's reward.

Beyond the ten components, one capability.

• thought_combination - konnectome now combines thoughts by minting Causalontology Causal Relation Objects through the reused causal_core. An atomic thought is an occurrent; a combination is a content-addressed causal_relation_object over occurrent causes and effects; one combination's effect reused as another's cause is a chain of thought, and a branching, converging set is a tree of thought. This is the substrate the Second Commandment names, and it touches nothing in the frozen ontology.

The Causalontology one-hundred-nineteen-vector conformance suite passes 119 of 119. Zero walls were hit; two observations were logged (not routed). PART FOUR - OTHER CONTEXT YOU SHOULD KNOW


• How to check the build is green. From the konnectome repository root: bin/run_pack_tests.sh <pack_name> runs one pack's tests; bin/check_pack_naming.sh checks the whole-word naming rule; bin/check_layers.sh checks the strict layer rule; bin/check_layer_binding.sh checks the layer-to-stratum binding. Each prints a clear pass-or-fail and exits zero when clean.

• The tooling. SWI-Prolog version 9.0.4 is installed. konnectome reuses PrologAI, it does not fork it: the bin scripts put PrologAI's packs on the SWI-Prolog library path (via PROLOGAI_HOME, defaulting to /home/ccaitwo/PrologAI), so a konnectome pack can load library(causal_core) and the like from PrologAI directly. konnectome's own packs live under packs/.

• The build-against commits, for reproducibility. causalontology at git head 98ebb33 (frozen Causalontology 3.0.0). PrologAI at git head 0fed5d1, tag prologai-wave10-complete. Mentova at git head fb7d48f.

• The versioning discipline (do not break it). Every code change bumps the six SPARCD documents in lockstep AND the tutorial, each gaining a Realization Log entry, with the superseded versions moved into docs/archive/ via git mv in the same change (Ninth, Tenth, and Fifteenth Commandments). The ledger and the README are updated in the same change (Eighth and Eleventh Commandments). Work on a feature branch, open a pull request, squash-merge your own and delete the branch, never push directly to main, and never credit an artificial-intelligence tool as an author (Fourteenth Commandment).

• The two logged observations. First, the Causalontology specification file spec/identity.md still says "seventeen whole-word schemes" and omits the eighteenth kind from one inline list, though the rest of the specification carries all eighteen - a stale count, logged for the owner, not routed. Second, a Causalontology token_occurrence requires an absolute timestamp while a konnectome tick is dimensionless-ordinal; the observer resolves this with a nominal, reproducible simulated clock, and the 3.0.0 ordinal tick unit remains available for native ordinal windows later. Neither is a wall.

• Honest limits of what runs. The sensorimotor loop is minimal: the body model is a single move-toward-set-point step, action proposals come only from the drives, and multi-tick interface delay lines, computational-interface transforms, homeostatic scaling, the eligibility trace, and representing interfaces as Causalontology conduits are all recorded in the ledger as pending refinements, not walls. The developmental ladder of Appendix 6 (senses, holding an object in mind, language, reasoning, emotion, embodiment) is the open research road ahead, and it rests on the running loop now in place.
