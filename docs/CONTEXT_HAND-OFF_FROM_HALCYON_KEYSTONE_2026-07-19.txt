PART THREE - THE FROZEN DATA-STRUCTURE STANDARD (Causalontology)


Causalontology is the data-structure language every part of the system speaks (it is a process-first foundational ontology - a shared vocabulary of causes, effects, and the structures that relate them, with each record identified by a content-addressed hash, meaning the identifier is derived from the record's own content).

Its current version is Causalontology 3.0.0, and it is FROZEN.

Do not change the ontology inside konnectome.

If konnectome discovers a data-structure gap, route it to the held change order named Causalontology_4_0_0_CHANGE_ORDER_v1 and to a future konnectome_ledger - do not patch the ontology in place.

PrologAI declares and passes the Causalontology 3.0.0 conformance suite: all 119 vectors (labeled V01 through V119), vendored under tests/causalontology_conformance/, run by bin/run_causalontology_conformance.sh.

The reusable conformance engine lives in the causal_core pack.


PART FOUR - WHAT konnectome INHERITS (the finished language)


These are the constructs Wave 10 delivered.

They are all at Layer 0 (base infrastructure) unless noted, and all are pack-qualified whole-word modules you can load and reuse.

Six brand-new packs.

- affective_state (Work Package 430): a first-class, persisted, modulatory affective state - a held valence, salience, mood, and cortisol tone that survives across calls and derives a baseline-or-stress regime that later processing reads as context.

- tick_scheduler (Work Package 432): a Lattice-backed deferred-reactivation construct on ordinal ticks - it schedules and enacts a future reactivation counted in dimensionless ordinal ticks, never in wall-clock seconds (a wall-clock unit is refused).

- managed_seam (Work Package 433): a first-class managed cross-stratal seam - it records a jump across non-adjacent strata with an honest-ignorance mechanism status (absent, unmodeled, or modeled), an optional drawn chain checked well-formed, a checkable home rule, and a queryable runtime event.

- realization (Work Package 434): a structure-to-dynamics binding - it binds a grounded structure record to the native law or typed signal that realizes it, by identity, and the binding is checkably real (a missing realizer is reported as a finding, not a shared English word).

- packaging (Work Package 436): dependency kinds (structure-only versus runtime), loadable pack faces, a facade or bundle, and a cross-pack record registry.

- coordination (Work Package 437): ergonomic coordination affordances for the single-threaded reentrant-loop model - a journal-free store with a keyed await, an ordered channel, a bounded loop driver, a reentrant-loop descriptor, a runtime layer-aware transport that refuses an upward send, and a glass-box hop trace.

Four extended packs.

- membership_contract (Work Packages 431 and 438): a runtime postcondition that an output must be a member of an input set - extended with a context-aware accessor, a purity guard, a determinism check, and a find-first-member filtering mode.

- layer (Work Packages 435 and 438): the strict layer rule (a lower layer may not depend on a higher one) - extended with cross-repository reach (a shared global coordinate via a per-repository offset, a unioned scan, a packs-directory argument on the checker) and intra-pack reach (a sub-module layering, coupling, and testability check), plus a binding-freshness check.

- lattice (finding N1): the shared persistent memory store - its transaction predicate is now correctly declared a meta-predicate.

- causal_core (Work Package 429): the Causalontology vocabulary engine - it adopted Causalontology 3.0.0, adding the ordinal tick unit, the eighteenth kind (the cross-stratal seam) with its well-formedness and home rules, and the conduit realized-by field.

Everything above is load-bearing, tested, and gated in Continuous Integration (the automated checks that run on every change).


PART FIVE - THE STANDING RULES YOU MUST FOLLOW


These rules are non-negotiable and apply to konnectome from its first commit.

- Naming convention: every pack name and every predicate name is lowercase snake_case built from whole English words, never abbreviations and never un-underscored concatenations; every predicate is prefixed with its pack's full whole-word name; terse two-letter prefixes are banned; the one branded abbreviation Causal Relation Object is spelled out in full as causal_relation_object everywhere, in code and prose alike.

- English Readable Code (ERC): every single line of code in every Prolog file carries one plain-English comment on the line immediately above it - modules, tests, manifests, and code quoted in papers included.

- The Specification, Pseudocode, Architecture, Refinement, Completion methodology (SPARC): every code change increments the affected SPARC document volumes in docs/, recording what changed and why, with a footer naming all five new versions; pure tooling or documentation-governance changes are the recognized exception and do not bump SPARC.

- The six-file repository documentation rule: every repository that contains functional code carries, from its first commit, a docs/ folder with six versioned files named for the repository - a Specification, a Pseudocode file (written in ERC), an Architecture, a Refinement, a Completion, and a Demonstration; konnectome will need this set on day one.

- Branch and pull-request rule: every unit of work uses a feature branch and a pull request; nothing is pushed directly to the main branch; a pull request you authored is merged immediately with a squash merge that deletes the branch; you do not auto-merge pull requests opened by anyone else.

- Attribution rule: no artificial-intelligence tool is ever credited as an author or co-author anywhere - no co-authored-by lines in commits, no such references in source or papers; element names are original, and no source-origin terms are introduced.

- Archive rule: only the latest version of any versioned document lives in docs/; the moment a new version is made, the superseded one is moved into docs/archive/ in the same change, and stragglers are swept.

- The tutorial, README (the repository's front-page description file), and Pack Reference update rules: a new pack earns a tutorial chapter and index entry, a README table row and narrative update, and a Pack Reference entry listing its public predicates - all in the same change as the code.

- Regression honesty: there is a fast ten-percent mini-regression gate (run by bin/run_mini_regression.sh, which reports 40 of 40 and 12 of 12) and a slow full regression; a green mini-run is never rounded up to the full 400 of 400 or 120 of 120 claim - only a genuine full run refreshes those claims, and the outstanding debt is tracked in REGRESSION_DEBT.md.

- Work Package and Layer numbering: Work Package numbers and Layer numbers are one shared global counter; take the next free numbers and never reuse them.

- Style: never use Roman numerals; use Arabic numerals or spelled-out words.

PART SEVEN - LESSONS AND SHARP EDGES I HIT (so you do not)


- PLUnit module qualification: when a test file declares itself a module, a goal you pass into a construct's meta-predicate is qualified with the TEST module, not the user module; if you guess wrong the goal is reported as an unknown procedure; qualify meta-goals explicitly (this bit me twice, in the temporal stage and the refinements stage).

- The full ARC benchmark is slow: the isolated entry point arc_benchmark_run over all 400 tasks takes more than ten minutes, and the booted driver run is similar; run these as background jobs, not foreground, and do not run two heavy Prolog benchmarks at once - they contend for the processor and both time out.

- The full regression is run by feeding a manifest of every task identifier through the mini-regression driver (which uses the same solving core), and separately by the isolated arc_benchmark_run entry point; both must show an empty fail list.

- The test runner bin/run_pr_tests.sh was fixed during close-out to classify a suite by its exit code rather than by a fragile text match, because a single-test suite prints "test passed" (singular) which the old text match missed; if you write single-test suites, this is now handled correctly.

- The file CLAUDE.md at the PrologAI repository root is git-untracked (local only); it holds the current-state narrative and the current SPARC versions, and you update it in the same change as code, but it does not travel in a pull request.

- Frozen means frozen: the ARC solving core, the Causalontology 3.0.0 standard, the earlier spike, the three organizational arms, and the built brain regions were all treated as read-only during Wave 10; where a workaround was needed it was demonstrated on the PrologAI side rather than by editing a frozen thing - carry that discipline into konnectome.
